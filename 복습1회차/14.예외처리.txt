Throwable
Error - 시스템에서 발생하는 에러 -> 통제 불가능

Exception  - 예외 -> 프로그램상에서 통제 가능
- RuntimeException 아닌 Exception 
	- 컴파일시에 체크되는 예외 
    - 예외 발생 -> 컴파일 X
	- IOException 

- RuntimeException 
    - 실행 중에 체크되는 예외
	- 컴파일은 O -> 실행중에 예외 체크 -> 프로그램 중단 
	

예외 발생 -> 프로그램 중단

1. 예외 발생시 프로그램 중단 방지
2. 예외에 대한 후속 처리(catch)  - 예외에 대한 추가 로직, 기록
try {
	// 예외가 발생할 수 있는 구간 
	
} catch ( 예외 클래스 e) {
 // 예외 처리 부분 
} finally {
	// 예외가 발생하든 안하든 무조건 실행되는 영역 
}

try - with - resources
try (.... 리소스를 자동으로 닫을 인스턴스; 인스턴스......) {

} catch (......) {

} 

AutoCloseable 인터페이스의 close() 메서드를 구현한 인스턴스는 -> 자동으로 close() 메서드가 try 구간이 끝나는 부분에서 호출


throws -> 예외 전가 -> 메서드를 호출하는 쪽에서 예외처리를 전가
