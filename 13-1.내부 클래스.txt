내부 클래스란?
	클래스 내부에 클래스가 정의 되어 있는 것 

1. 인스턴스 내부 클래스 - 멤버변수(인스턴스 변수, 인스턴스 메서드)

jdk15 이하 - 내부에서 static 변수, 메서드 X
			   -> 내부클래스의 인스턴스 생성을 하려면 외부 클래스의 인스턴를 먼저 생성 
	
jdk16 이상 - static 변수, 메서드 
			   -  데이터 영역에 내부 클래스더라도 클래스 파일이 올라갈때 처리
			   

2. 정적 내부 클래스 - 정적 변수, 정적 메서드
	- 외부 클래스의 인스턴스 생성과 관련없이 독자적으로 생성 가능
	- 일반 외부 클래스와 거의 동일(외부 클래스를 통해서 접근)

	참고)
	정적 변수, 정적 메서드 -> 인스턴스 생성 X -> 클래스 명으로 접근
	인스턴스 생성 이전 부터 이미 초기화 -> 사용 가능 

3. 지역 내부 클래스 - 메서드 안에 정의되어 있는 클래스
		- 지역 내부의 클래스의 인스턴스는 메서드가 호출 -> 스택에 올라가야 접근 가능 
		- 지역 내부 클래스 외부에 있는 메서드 안에  정의되어 있는 지역변수 -> 상수(public final)
		- 메서드 호출 -> 인스턴스 반환 -> 종료(스택에서 제거)
		- 메서드의 지역변수를 계속 접근 -> 상수화(데이터 영역)


4. 익명 내부 클래스
	인터페이스, 추상클래스 -> 인스턴스 생성 X 
									-> 추상 메서드의 구현 부분이 구현이 완료 X
									
	지역 내부 클래스 형태로 추상 메서드의 미구현 부분을 정의 함으로써 인스턴스 생성
